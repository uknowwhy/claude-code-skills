# 视频逐字稿格式化工具 V3

## 🚀 重大更新（V3.0）

基于真实项目经验重构，全新升级：
- ✅ **智能语义分段** - 基于话题转换、序列词、场景变化自动分段，每段80-100字
- ✅ **视觉呼吸原则** - 符合现代移动阅读习惯，营造舒适阅读节奏
- ✅ **口误智能识别** - 删除自我纠正、重复表达、说话人标签
- ✅ **语义完整性保护** - 不在句子中间切断，保持逻辑完整
- ✅ **零规则副作用** - 避免误删顿号、列表项等正常内容

## 概述

将带时间戳的视频逐字稿自动转换为格式规范、段落舒适、可读性强的最终文稿。

**核心特性:**
- **智能语义分段** - 识别话题转换点（"接下来"、"但是"、"更重要的是"）
- **80字黄金段落** - 符合移动阅读习惯，营造视觉呼吸感
- **口误自动删除** - 删除重复表达、自我纠正等口语问题
- **说话人标签清理** - 完全移除"说话人1"等标签（含句中位置）
- **英文单词修复** - 修复被空格错误分割的英文单词
- **被切断句子合并** - 合并被空行错误分开的完整句子

## 使用场景

✅ **适用场景:**
- 视频课程逐字稿整理
- 演讲/访谈/教学视频后期处理
- 播客/音频节目文字稿整理
- 需要保持口语感的整理类内容

⚠️ **不适用场景:**
- 文学创作（需保留原汁原味）
- 法律文书（不能改原意）
- 已经过人工精校的文稿

## 🎯 完整工作流程

```
原始逐字稿（含时间戳、说话人标签）
  ↓
步骤1: 基础清理（去除时间戳、元数据、说话人标签）
  ↓
步骤2: 口误修复（删除自我纠正、重复表达）
  ↓
步骤3: 句子合并（修复被错误切断的句子）
  ↓
步骤4: 智能语义分段（80字/段，保持语义完整）
  ↓
步骤5: 人工精修（处理机器无法理解的内容）
  ↓
最终成品
```

## 📋 V3新增功能详解

### 1. 智能语义分段

**核心原则:**
- **视觉呼吸**: 每段80-100字，避免大段文字压迫感
- **语义完整**: 不在句子中间切断
- **话题切换识别**: 在转换词处自然分段
- **序列词识别**: 在"第一"、"第二"等处分段

**分段信号词:**

强分段（必须分段）:
- "还推荐一个"
- "更重要的是"
- "最后我想"
- "好啦，"

中等分段（优先考虑）:
- "首先"、"接下来"、"然后，"、"现在，"
- "这个时候，"
- "第一。"、"第二"、"第三"...
- "但是，"、"不过，"、"然而，"

**分段示例:**

```
原文（超长段，无换行）:
你好今天我们要讨论一个非常重要的话题关于人工智能的发展趋势首先让我们来看一下历史背景人工智能的发展可以分为几个阶段第一个阶段是符号主义第二个阶段是连接主义第三个阶段是深度学习时代现在我们正处于第四个阶段也就是大语言模型时代...

V3智能分段后（80字/段）:
你好，今天我们要讨论一个非常重要的话题，关于人工智能的
发展趋势。首先让我们来看一下历史背景。人工智能的发展可以
分为几个阶段。

第一个阶段是符号主义，第二个阶段是连接主义，第三个阶段是
深度学习时代。现在我们正处于第四个阶段，也就是大语言模
型时代...
```

**效果对比:**

| 版本 | 段落数 | 平均长度 | 视觉感受 |
|------|--------|----------|----------|
| 原版 | ~8段 | ~600字 | 密集压抑 |
| 200字分段 | 15段 | 180字 | 较舒适 |
| **V3 (80字)** | **50段** | **90字** | **轻盈透气** |

### 2. 口误智能识别

**识别的口误类型:**

1. **自我纠正**
   ```
   原文: 我今天想讲的，我今天想讲的是关于机器学习的基础
   修复: 我今天想讲的是关于机器学习的基础
   ```

2. **重复表达**
   ```
   原文: 我们首先需要，我们首先需要明确一点
   修复: 我们首先需要明确一点
   ```

3. **结巴/犹豫**
   ```
   原文: 这个模型。这个模型非常强大
   修复: 这个模型非常强大
   ```

4. **说话人标签**
   ```
   原文: 我们可以开始。说话人 1\n\n接下来讨论第二个问题
   修复: 我们可以开始。接下来讨论第二个问题
   ```

### 3. 被切断句子合并

**问题模式:**
```
原文: 当你点击这个按钮之后你会。\n\n发现界面发生了很大变化
修复: 当你点击这个按钮之后你会发现，界面发生了很大变化
```

**合并规则:**
- 句子不以句号结尾，但被空行分开
- 下一行以小写字母或动词开头
- 合并时保持语义流畅

### 4. 英文单词修复

**常见问题:**
```
"Git Hub" → "GitHub"
"Deep Seek" → "DeepSeek"
"USB c" → "USB-C"
```

### 5. 语义完整性保护

**避免的错误:**

❌ **错误做法（机械替换）:**
```python
# 规则过于宽泛，误删正常内容
text = text.replace('问题一，问题二', '问题二')
# 结果: "问题一、问题二、问题三" → "问题二、问题三" ❌
```

✅ **V3改进:**
```python
# 只删除明确的口误，保留所有不确定的内容
# 保护标点符号、列表项、引用内容
```

## 🎓 核心设计思想

### V3的核心改进

**从"机械规则"到"语义理解"**

V2的问题：
```python
# 机械替换，不考虑上下文
text = text.replace('问题一，问题二', '问题二')
# 结果: 可能误删列表项中的顿号
```

V3的改进：
```python
# 只删除明确的错误，保留所有不确定的内容
# 保护标点符号、列表项、引用内容
# 基于语义而非字符串匹配
```

**段落划分的进化**

V1/V2: 机械按句号分段，不考虑语义
```
❌ "点击之后你会。\n\n发现界面有变化"
   → 在句号处分段，破坏语义
```

V3: 语义完整性优先
```
✅ 识别到未说完，合并
   → "点击之后你会发现，界面有变化"
```
