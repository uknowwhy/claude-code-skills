# 逐字稿人工修正清单

## 一、机器无法自动处理的问题类型

### 1. 断句错误（需重连句子）
**特征：** 句子中间出现句号，导致语义破碎

**典型案例：**
```
原句：然后他会对于。跟你的这个交互，跟外界的交互它都会有记忆
修正：然后他会处理跟你的交互，跟外界的交互都会有记忆

原句：我在这里。输入Skills，这个是他说可以把所有...
修正：我在这里输入Skills，他可以把所有...
```

**处理方法：**
- 删除中间句号
- 调整语序，保持语义完整
- 必要时添加连接词

---

### 2. 语境判断（需看上下文）
**特征：** 同样的表达，在不同语境下需要不同处理

**案例A：指代不明**
```
问题：它/他/她
- 指 AI 系统 → "它"
- 指人（导师、朋友、专家） → "他"
- 需看上下文判断
```

**案例B："这个/那个"的保留**
```
删除：这个就是那个AI Agent → 这就是AI Agent
保留：这个Skill用起来很方便（强调特指）
```

**处理方法：**
- 理解指代对象
- 判断是否需要强调

---

### 3. 语序调整（需理解语义）
**特征：** 句子语法正确但表达不自然

**典型案例：**
```
原句：AI也是需要你给它这样的一些手把手的传授
修正：AI也需要手把手的传授

原句：这实际上是把荣宁二府金陵十二钗都放在里面了
修正：这实际上是把荣宁二府和金陵十二钗都放在里面了

原句：然后他会对于跟你的这个交互
修正：然后他会处理跟你的交互
```

**处理方法：**
- 提取主干
- 删除冗余修饰
- 调整语序符合中文习惯

---

### 4. 漏词补全（需理解语境）
**特征：** 口语中省略了某些词，书面语需要补上

**典型案例：**
```
原句：这实际上是把荣宁二府金陵十二钗
修正：这实际上是把荣宁二府和金陵十二钗

原句：前些日子得到发了一个奖牌
修正：前些日子得到平台给我发了一个奖牌
```

**处理方法：**
- 根据语境补全逻辑连接词
- 补充明确指代的名词

---

### 5. 专业表达统一（需领域知识）
**特征：** 术语有多种表达方式，需要统一

**案例：**
```
Claude Code vs Claude Code
- 保持统一，不要出现 Claude-code、CLAUDE CODE

Skill vs skill
- 作为专有名词时大写：Skill
- 作为普通名词时小写：技能
```

**处理方法：**
- 建立术语表
- 全文统一替换

---

### 6. 逻辑连接（需理解文意）
**特征：** 句子之间缺乏逻辑连接，读起来突兀

**典型案例：**
```
原句：我比较推荐的是这个叫42 Plugin。这个是由活水智能团队做出来的...
修正：我比较推荐的是42 Plugin，它是由活水智能团队做出来的...
```

**处理方法：**
- 添加连接词：它、这、而、但
- 调整句子结构，让过渡更自然

## 二、人工修正优先级

### 高优先级（必须修正）
1. **断句错误** - 严重影响阅读
2. **明显的漏词** - 导致语义不清
3. **指代错误** - "它/他"混淆

### 中优先级（建议修正）
1. **语序调整** - 提升可读性
2. **逻辑连接** - 让行文更流畅
3. **专业表达** - 保持一致性

### 低优先级（可选修正）
1. **语气微调** - 保留讲师个人风格
2. **细节优化** - 不影响理解的表达

---

## 三、人工修正工作流

### 步骤1：通读全文
- 标记读起来不顺的地方
- 记录明显的问题

### 步骤2：分类处理
- 按"问题类型"分类
- 从高优先级开始修正

### 步骤3：交叉检查
- 修正后再通读一遍
- 确保没有引入新问题

### 步骤4：记录经验
- 将新发现的问题补充到规则模板
- 更新处理脚本
