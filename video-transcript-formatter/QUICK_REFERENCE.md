## 🚀 快速开始

```bash
# 1. 准备文件
cp original.txt transcript_final_v3.txt

# 2. 运行自动处理
python3 smart_refine_transcript_v2.py

# 3. 人工精修
# 打开 transcript_20250202_refined_v2.txt
# 参考 manual_fixes_checklist.md 逐段检查

# 4. 输出成品
# 保存为 transcript_final.txt
```

---

## 📋 核心规则速查

### 1️⃣ 冗余套话（立即删除）
```
我们打一个比方 → 打个比方
你好比说 → 比如
实际上 → 删除
在这里 → 删除
```

### 2️⃣ 重复"说"（立即删除）
```
说这个 → 删除
说你 → 删除
说是 → 是
```

### 3️⃣ 标点统一
```
。。→ 。
，，→ ，
.，→ 。（中文用中文标点）
```

### 4️⃣ 专业术语
```
USB c → USB-C
Git Hub → GitHub
非常的 → 非常
```

### 5️⃣ 冗余动词
```
去执行 → 执行
来调整 → 调整
来看 → 看
```

## ⚠️ 必须人工检查的问题

### 🔴 高优先级
- [ ] 断句错误："然后他会对于。跟你的" → "然后他会处理跟你的"
- [ ] 漏词："荣宁二府金陵十二钗" → "荣宁二府和金陵十二钗"
- [ ] 指代错误：AI用"它"，人用"他"

### 🟡 中优先级
- [ ] 语序调整："它就是那个" → "它就是"
- [ ] 逻辑连接：添加"它"、"这"、"而"等连接词

## 💡 记住这几点

1. **不要追求100%自动化** - 机器处理80%就很好了
2. **每次都记录新问题** - 逐步完善规则库
3. **保留所有版本** - 便于对比和回溯
4. **先小范围测试** - 确认效果再批量处理
5. **保留讲师风格** - 不要过度修正

## 🆘 遇到问题？

**Q: 脚本报错？**
A: 检查文件路径是否正确，确保输入文件存在

**Q: 处理后反而更乱？**
A: 可能规则冲突，检查 `transcript_rules_template.md` 中是否有重复规则

**Q: 某些规则没生效？**
A: 可能是上下文不匹配，需要人工处理或调整规则

**Q: 如何添加新规则？**
A: 编辑 `transcript_rules_template.md`，然后更新 `smart_refine_transcript_v2.py`
